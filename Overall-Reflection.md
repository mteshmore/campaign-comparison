## Reflection and Areas for Improvement

### Responsive Design Expansion

I wasn’t able to fully design for multiple screen sizes, but the current layout is structured in a way that supports responsive behavior using flexbox. The components can reflow naturally as the screen size changes. With more time, I would be more intentional about defining what information should be hidden, condensed, or prioritized at tablet and mobile breakpoints to ensure the most important campaign details remain clear and accessible.

### Front-End Implementation and Visual Polish

All required functionality from the brief is implemented, but I did not fully refine the visual design or branding. If I had more time, I would standardize card, font, and button sizing and apply a consistent color scheme. I would also add helper text for key campaign attributes to reduce confusion and improve clarity. Additionally, I would refine interaction details, such as smoother transitions when adding or selecting campaigns, to make the experience feel more seamless and intentional.

### Testing Improvements

I would expand test coverage by writing more test cases, including both happy path scenarios and validation error states, to better ensure backend reliability and edge case handling.

### Data Structure Optimization

Lastly, I would restructure the data setup by separating mock data into its own file instead of embedding it directly in the front-end. This would improve maintainability, scalability, and overall project organization.

### Time Management and Prioritization

The backend required more work than initially expected, and I chose to invest extra time in refining the wireframe to ensure the main views were clear and functional. In the future, I would finalize a minimal wireframe first and then focus on making the front-end visually polished. This would also leave more time to expand backend features and improve collaboration.  

Throughout the project, I prioritized the core elements: main views for design, essential front-end functionality, and the API request. This approach ensured the project met key goals while leaving room for enhancements if more time were available.
